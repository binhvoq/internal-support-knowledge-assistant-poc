# Policy PDF RAG Ingestion (Azure Indexer + Chunk Index)

## Requirement

Admin can upload internal policy PDFs into the knowledge base. The AI assistant must later use those documents for RAG when users ask questions in ticket auto-suggestion or chat. The admin UI should show whether each uploaded file has been indexed into searchable chunks and is ready for advisory answers.

## Architecture (current)

**App responsibilities**

- `Knowledge Admin` UI: upload PDF, title/category, status, delete.
- `KnowledgeService`: store document metadata in SQL Server, upload **original PDF** (or `.txt` for manual text docs) to Azure Blob with metadata, trigger Azure AI Search indexer, poll chunk visibility, expose `/search` for hybrid retrieval.
- `AiOrchestrator`: query `/search`, inject **top chunks** (not full PDF) into LLM prompts.

**Azure responsibilities**

- Blob container `knowledge-docs`: source files at `{documentId}/{fileName}` with metadata (`documentid`, `title`, `category`, `filename`, `uploadedat`).
- Azure AI Search index `knowledge-chunks`: **one row per chunk** (`chunkId` key).
- Skillset `knowledge-pdf-skillset`:
  1. built-in PDF/text extraction from blob,
  2. Text Split skill (overlap, ~1000-token equivalent page length),
  3. Azure OpenAI Embedding skill per chunk,
  4. index projections → `knowledge-chunks`.
- Indexer `knowledge-blob-indexer`: scheduled every 5 minutes + on-demand run after upload/reindex.

App **does not** use PdfPig or app-side document embedding for PDF ingestion anymore. `EmbeddingService` remains only for **query** embeddings at search time.

## Endpoints

| Method | Path | Notes |
|--------|------|-------|
| POST | `/documents/upload-pdf` | Upload PDF to Blob → `Processing` → trigger indexer → poll chunks → `Ready`/`Failed` |
| POST | `/documents` | Text doc: save DB + upload `.txt` to Blob → same indexer pipeline |
| DELETE | `/documents/{id}` | Delete chunks in Search → delete Blob → delete DB row |
| POST | `/documents/reindex` | Run Azure indexer (no app-side embedding) |
| GET | `/search` | Hybrid/vector/keyword against `knowledge-chunks` |

## Chunk index schema (`knowledge-chunks`)

| Field | Role |
|-------|------|
| `chunkId` | Key (generated by index projections) |
| `documentId` | Parent document, filterable |
| `title`, `category`, `fileName` | Metadata from blob |
| `sourceUrl` | Blob path |
| `parentId` | Parent blob key do Azure index projection populate (khong phai chunk key) |
| `content` | Chunk text (searchable) |
| `embedding` | 1536-dim vector (`text-embedding-3-small`) |
| `uploadedAt` | From blob metadata |

## Status model

- `Processing`: blob uploaded, waiting for indexer chunks.
- `Ready`: indexer run success **va** co chunk cho `documentId`.
- `Processing`: indexer van chay hoac chua xac nhan du chunk trong timeout; `DocumentIngestionRefreshBackgroundService` tiep tuc poll moi 30s.
- `Failed`: indexer transientFailure hoac loi pipeline; xem `ingestionMessage`.

## RAG flow

1. Admin uploads PDF in `Knowledge Admin`.
2. App stores metadata + uploads PDF to Blob.
3. App triggers indexer and polls `documentId` chunk count.
4. User question → hybrid search on `knowledge-chunks` → top chunks in prompt.
5. LLM answers only from chunk context; if no relevant chunk, says insufficient information.

## Delete flow

1. Delete all chunks where `documentId eq '{id}'` from Azure AI Search.
2. Delete blob `{documentId}/{fileName}`.
3. Delete SQL Server document row.

If chunk deletion fails, DB row is kept.

## Demo PDFs

Local demo PDFs under `test-pdfs/` (gitignored). On startup, if Azure pipeline is configured and `device-replacement-policy.pdf` exists, the service auto-uploads it through the indexer pipeline.

## Limitations

- Text-selectable PDF only (no OCR).
- Indexer poll timeout defaults to 120s (`AzureSearch:IngestionPollTimeoutSeconds`).
- Legacy index `knowledge-documents` (1 PDF = 1 vector) is no longer used by the app.

## Configuration

```json
"AzureSearch": {
  "IndexName": "knowledge-chunks",
  "DataSourceName": "knowledge-blob-datasource",
  "SkillsetName": "knowledge-pdf-skillset",
  "IndexerName": "knowledge-blob-indexer",
  "ChunkMaxPageLength": 1000,
  "ChunkPageOverlapLength": 200
}
```

Skillset/indexer are provisioned at `KnowledgeService` startup via `AzureSearchIngestionService` (REST for skillset index projections + SDK for datasource/indexer).
